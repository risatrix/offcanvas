What This Plugin Does

Targets a menu, builds an offcanvas version that hides behind the rest of your page, and gives you the ability to show or hide it based on toggling classes. You call the plugin like this:

(targetMenu).offcanvas();

where targetMenu is any jQuery selector.

NB: You have to add the toggle to your code yourself. The default plugin expects .nav-toggle but you can pass in your own selector as an option. Make the browser window narrow (i.e. mobile sized) to see the toggle on this page.

What This Plugin Does NOT Do

This plugin does NOT animate via jQuery. It builds a menu and adds classes that the CSS responds to, allowing for progressive enhancement. Read David Bushell's excellent article for an explanation of why this is the best approach.

Use Cases

1) You have a pre-existing menu and want to make it offcanvas for mobile - just make sure to only show the toggle at the right size and includ media queries in your CSS.

2) You want to shift your content around with a toggle at any size. Great. Just show the toggle and don't constrain the CSS with media queries.

-You want to do something crazy we haven't even thought of yet. Good news, everyone: once you're toggling of classes you can do whatever you want with the CSS - transitions, flips, leaps, bounds!

What's with Me Having to Mess with the CSS?

This is just a basic, no-frills, right-sliding menu. It's not one-size-fits-all. It doesn't make sense to clutter up the js with left or right classes when the entire point of this approach is to separate the js from the styles. This also allows you great flexibility in adapting the menu to your needs. Check out this article on sidebar transitions for inspiration.

Transform FTW!

The only way to achieve smooth menu transitions on mobile is with the CSS3 Transform/Translate3d (cf. this article). property, because that is the only hardware-accelerated property that works. The default CSS uses this if the capability is detected, but falls back to absolute positioning if not.
